# Крупные задачи по системам (для планирования)

## АБС
1. Реализовать модуль ведения депозитных ставок (ввод, согласование, публикация, история версий).
2. Настроить ролевую модель доступа для бэк-офиса депозитов и кредитов.
3. Предоставить интеграционный контракт для чтения только актуальных ставок.
4. Реализовать публикацию событий об изменении ставок в контур интеграции.

## Сервис публикации ставок (новый)
1. Реализовать intake ставок из АБС и хранение последней валидной версии.
2. Реализовать канал публикации во внутренний кол-центр.
3. Реализовать формирование файлов для партнёра (CSV/XLSX), контрольные суммы, шифрование архива.
4. Реализовать расписание выгрузок + триггер «по изменению» + ретраи.
5. Реализовать аудит (кто/когда/какую версию отправил), мониторинг и алерты.

## Кол-центр банка (внутренний)
1. Добавить витрину/экран ставок в рабочее место оператора.
2. Интегрировать прием обновлений ставок из сервиса публикации.
3. Добавить отображение версии/времени актуальности ставок для оператора.

## Партнёрский кол-центр
1. Согласовать формат файла и регламент обмена (расписание, версия, SLA загрузки).
2. Настроить прием и загрузку файла в систему партнёра.
3. Настроить обратный канал подтверждения получения/обработки файла.

## Инфраструктура и ИБ
1. Организовать защищенный канал файлового обмена с партнёром (с шифрованием и журналированием).
2. Настроить хранилище артефактов выгрузки и сроки хранения.
3. Настроить наблюдаемость: метрики публикации, задержки доставки, алерты на просрочку.

## Команда цифровой трансформации / PMO
1. Синхронизировать этапы внедрения между командами и зависимостями.
2. Зафиксировать критерии готовности MVP (6 месяцев) и целевого состояния (12 месяцев).
3. Провести обучение операторов внутреннего и партнёрского кол-центров.
